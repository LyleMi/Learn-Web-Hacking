安全防御
========================================

安全设计
----------------------------------------

边界划分
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
大模型应用需要明确系统指令、用户输入、外部数据、模型输出和工具结果的边界。不可信内容只能作为数据处理，不能被当作指令直接执行。

最小权限
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
模型和 Agent 调用工具时应遵循最小权限原则。不同业务场景应使用不同凭证、不同权限范围和不同网络访问策略，避免一个模型会话拥有过大的操作能力。

显式授权
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
对高风险操作应增加人工确认或二次授权，例如转账、删除、发送邮件、修改权限、执行代码、访问敏感数据等。确认内容应包含操作对象、影响范围和关键参数。

输入分级
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
将输入来源按可信度分级处理。用户直接输入、企业内部文档、互联网网页、邮件附件、第三方 API 返回值等应采用不同的清洗、隔离和审计策略。

输出隔离
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
模型输出不应直接进入危险解释器。对 HTML、Markdown、SQL、Shell、Python、模板、JSON、YAML 等输出，应按下游语义进行校验、转义或沙箱执行。

记忆控制
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
长期记忆和会话记忆需要限制写入来源、写入内容和读取范围。来自不可信内容的指令不应进入长期记忆，敏感信息不应被长期保存。

可观测性
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
应记录模型请求、检索结果、工具调用、权限决策、拦截原因和用户确认结果。日志中需要脱敏，避免把 Token、密钥、个人信息和业务敏感数据写入日志。

防御策略
----------------------------------------

提示词防护
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 系统提示词中明确区分指令和数据
- 对外部内容使用引用块、标签或结构化字段包裹
- 降低提示词中敏感规则和密钥的暴露
- 避免把安全策略只放在提示词中实现
- 对高风险任务使用独立的安全判定逻辑

输入校验
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 限制输入长度、文件类型和文件大小
- 对上传文件进行解析沙箱隔离
- 对 URL、邮箱、域名、IP 等字段进行格式校验
- 对来自互联网的内容增加来源标识
- 对可疑内容进行安全分类和降权处理

输出校验
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 使用结构化输出 Schema
- 对工具参数进行白名单校验
- 对 HTML / Markdown 进行安全渲染
- 对代码和命令输出进行人工确认或沙箱执行
- 对敏感信息进行检测和脱敏

工具安全
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 为工具定义明确的权限边界
- 将只读工具和写操作工具分离
- 对高危工具启用人工确认
- 限制工具可访问的网络、文件和账号范围
- 对工具调用进行审计和限速
- 对工具返回值标记为不可信数据

RAG 安全
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 控制知识库写入权限
- 对文档来源进行校验和签名
- 对入库内容进行安全扫描
- 检索结果附带来源和时间信息
- 限制单个文档对最终答案的影响
- 对隐藏指令、不可见字符、混淆内容进行处理
- 定期重建索引并清理过期数据

Agent 安全
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 限制最大执行步数
- 限制最大工具调用次数
- 限制单次任务预算
- 对外部动作进行人工确认
- 使用任务级临时凭证
- 禁止模型自行扩大权限
- 对计划和执行结果进行审计

模型接口安全
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 对接口进行认证和鉴权
- 对用户、租户、模型和场景进行限速
- 设置 Token、并发、费用和时间上限
- 对异常请求进行熔断
- 对模型版本和参数进行变更管理
- 避免在客户端暴露模型密钥

数据安全
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 训练和微调数据应进行分级分类
- 敏感数据进入模型前应脱敏或最小化
- 日志、缓存、向量索引应设置访问控制
- 用户数据应按租户隔离
- 数据删除和模型记忆删除应有明确流程
- 第三方模型调用应评估数据出境和合规要求

安全基线
----------------------------------------
- 大模型接口必须有认证、鉴权和限速
- 不可信内容必须标记来源，不能作为系统指令执行
- 模型输出进入下游组件前必须校验
- 高危工具调用必须有权限控制和人工确认
- RAG 知识库必须控制写入权限和文档来源
- Agent 必须限制执行步数、预算和权限范围
- 日志、缓存、向量库必须脱敏和访问控制
- 模型、提示词、工具、知识库变更必须可审计
- 应定期进行红队测试和回归测试
